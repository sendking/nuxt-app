### 2025年12月18日：实现动态日志页面功能

本次更新的目标是创建一个动态路由页面，用于展示 `changelogs/` 目录下的 Markdown 文件。

**核心变更：**

1.  **技术方案变更**：
    *   放弃了通过服务器 API 读取文件的旧方案。
    *   改为采用 Nuxt/Vite 内置的 `import.meta.glob` 功能，在构建时直接导入 Markdown 文件内容，更高效、更现代化。

2.  **项目结构与路由**：
    *   **纠正 `srcDir`**：将 `pages` 目录移动到 `app/` 目录下（`app/pages/`），以符合本项目 `app.vue` 所在的 `srcDir` 约定，解决了之前遇到的 `404` 路由找不到的问题。
    *   **日志列表页**：创建了 `app/pages/changelogs/index.vue`，它使用 `import.meta.glob` 扫描所有日志文件并生成一个导航列表。
    *   **动态详情页**：创建了 `app/pages/changelogs/[slug].vue`，它可以根据 URL 中的 `slug`（例如 `2025-12-18`）动态加载并渲染对应的 Markdown 文件内容。

3.  **配置与依赖**：
    *   在 `nuxt.config.ts` 中添加了 `pages: true` 和路径别名 `@changelogs`。
    *   重新安装了 `marked` 库用于解析 Markdown。

4.  **问题排查与修复**：
    *   修复了因在客户端代码中使用 Node.js `path` 模块而导致的 `500` 服务器错误。
    *   目前正在对 `import.meta.glob` 的路径解析问题进行诊断，以解决日志内容无法加载的问题。
